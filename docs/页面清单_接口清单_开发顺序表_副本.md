# CMMS 原型落地文档：页面清单 + 接口清单 + 开发顺序表

## 1. 文档目的

本文档用于将已完成的 CMMS（设备维护管理系统）前端原型，转化为可执行的开发方案，供前端、后端、联调与测试阶段直接使用。

核心目标：

- 明确本期开发范围
- 明确页面结构与模块边界
- 明确页面字段与接口映射关系
- 明确 Vue + SpringBoot 的推荐开发顺序
- 降低后续联调和返工成本

---

## 2. 本期建议开发范围

建议优先实现以下 P0 核心模块：

1. 登录与权限框架
2. 数据看板
3. 设备管理
4. 工单管理
5. 保养管理
6. 备件管理
7. 系统管理（基础版）

### 2.1 P0 / P1 / P2 优先级建议

#### P0（必须完成）
- 登录
- 系统基础布局
- 路由与权限控制
- 设备台账
- 工单主流程
- 保养工单
- 备件库存与预警
- 用户基础管理

#### P1（建议本期完成）
- 数据看板图表
- 保养模板管理
- 设备分类管理
- 出入库记录
- 通知中心

#### P2（可后续迭代）
- 二维码管理
- 更复杂的统计分析
- 移动端适配
- 审批流增强
- 多维报表导出
- 更细粒度的操作日志审计

## 5. 接口清单

以下接口以 RESTful 风格为建议，可按你的项目规范微调。

## 5.1 认证与权限

### 5.1.1 用户登录
- 方法：POST
- 地址：/api/auth/login

请求参数：
```json
{
  "username": "admin",
  "password": "123456"
}
```

返回示例：
```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "token": "xxxxx",
    "userId": 1,
    "username": "admin"
  }
}
```

### 5.1.2 获取当前用户信息
- 方法：GET
- 地址：/api/auth/me

### 5.1.3 获取当前用户菜单权限
- 方法：GET
- 地址：/api/auth/menus

## 第三阶段：核心业务模块
建议顺序如下：

1. 设备台账
2. 工单管理
3. 保养管理
4. 备件管理
5. 数据看板
6. 系统管理增强项

原因：
- 设备台账是主数据基础
- 工单是核心业务主流程
- 保养与备件依赖设备和工单逻辑
- 看板依赖业务数据沉淀后才有意义

---

## 9. 当前最建议你立刻去做的动作

按优先级排序如下：

1. 先确认本期只做哪些页面
2. 把每个页面字段列出来
3. 把每个页面按钮对应的接口列出来
4. 把工单状态流转图确定下来
5. 把数据库核心实体先定下来
6. 然后再让前端和后端并行开发

---

## 10. 一句话结论

原型完成之后，真正的下一步不是继续改视觉，而是把它整理成：

**页面清单 + 字段清单 + 接口清单 + 开发顺序表**

只有这样，这个原型才能从“展示稿”真正进入“可交付项目”阶段。
