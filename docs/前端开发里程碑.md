## 二、第一阶段（已完成）

### 2.1 工程与基建

- Vue 3 + Vite + TypeScript 项目初始化
- 公共请求封装：`src/utils/request.ts`（统一 `code` / `msg` / `data` 解包、错误提示、401 跳转）
- 按业务模块拆分 API：`src/api/`（auth, dashboard, equipment, workorder, maintain, spare, system, dict, role）

### 2.2 布局与导航

- 后台 Layout：可折叠侧边栏 + 顶栏面包屑 + 路由菜单
- 路由入口：`src/router/index.ts`（9 个业务路由 + 登录页）

### 2.3 列表页基建

- 列表页统一结构：筛选区 + 表格 + 分页
- `PageHeader` 公共组件
- 列表 Fallback 机制：`useListFallback` 在接口不可用时填充本地示例数据，避免白屏（受 `VITE_USE_LIST_FALLBACK` 控制）
- 字典加载：`useDictOptions` + `dictFallbacks.ts`，后端字典失败时走本地兜底

## 三、第二阶段（已完成）

### 3.1 登录与鉴权

- 登录页 → `POST /auth/login` → token + userInfo + roleCodes 写入 `sessionStorage`
- 请求头自动携带 `Authorization: Bearer <token>`
- HTTP 401 拦截：清理 token 并跳转 `/login`
- 路由守卫：未登录重定向、`meta.requiredRoles` 菜单级权限控制
- 开发模式可选 `VITE_DEV_BYPASS_AUTH` 跳过守卫

### 3.2 各模块 CRUD 完成度

| 模块 | 路由 | 已实现功能 |
|------|------|-----------|
| 数据看板 | `/dashboard` | KPI 指标卡 + ECharts 图表；`Promise.allSettled` 图表独立降级 |
| 设备台账 | `/equipment/list` | 列表 + 详情 / 新增 / 编辑 / 删除 |
| 维修工单 | `/workorder/list` | 列表 + 详情 / 报修 / 派工 / 关闭 |
| 备件管理 | `/spare/list` | 列表 + 详情 / 新增 / 编辑 / 删除 + 入库 / 出库 |
| 用户管理 | `/system/user` | 列表 + 详情 / 新增 / 编辑 / 删除 / 重置密码 |

前后端字段与路径对齐见 [`API_CONTRACT.md`](./API_CONTRACT.md)。

## 四、第三阶段（已完成）

### 4.1 保养工单全流程

| 能力 | 实现状态 |
|------|---------|
| 列表 + 筛选（关键字/状态/模板/日期范围） | 已完成 |
| 手动建单（选设备、选模板、计划日期、可选即时派工） | 已完成 |
| 大详情弹窗（基础信息 + 时间线 + 检查项 + 耗材 + 附件） | 已完成 |
| 派工（选执行人、计划执行时间、备注） | 已完成 |
| 执行（确认执行人、计划时间、开工备注） | 已完成 |
| 完成（结果选择 + 模板检查项 + 自定义检查项 + 备件选择 + 附件上传） | 已完成 |
| 关闭（原因分类 + 关闭说明，仅待派工/待执行可关闭） | 已完成 |
| 列表快捷操作（按状态显示派工/执行/完成按钮） | 已完成 |

### 4.2 保养规则管理

- 规则列表 + 新增 / 编辑 / 启停 / 手动触发生成
- 周期类型：DAILY / WEEKLY / MONTHLY / QUARTERLY / YEARLY
- 全局提前生成天数展示

### 4.3 保养模板管理

- 模板列表 + 新增 / 编辑 / 删除 + 详情（含检查项表格）
- `meta.requiredRoles` 限制仅管理员和保养主管可见

### 4.4 辅助模块

- **备件出入库记录** `/spare/stock-record`：列表 + 关键字/类型/时间范围筛选 + 分页
- **系统角色管理** `/system/role`：列表 + 新增 / 编辑 / 删除（仅管理员可见）

### 4.5 权限体系

四级角色优先级（`ROLE_ADMIN` > `ROLE_MAINTAIN_MANAGER` > `ROLE_EXECUTOR` > `ROLE_VIEWER`）：

- **路由级**：`meta.requiredRoles` 控制菜单可见性，无权限访问重定向到看板
- **按钮级**：保养模块所有操作按钮按角色 × 工单状态双重判定，禁用时显示 tooltip 原因提示
- **多角色处理**：取最高优先级角色为有效角色

## 五、第四阶段（进行中）

当前前端功能已全部实现，剩余工作集中在与后端联调对接和体验收敛。

| 类别 | 待办事项 | 优先级 |
|------|---------|--------|
| 联调验证 | 保养工单全流程端到端回归（派工→执行→完成→关闭，含附件和耗材） | 高 |
| 联调验证 | 字典接口 `/dict/{type}` 与后端最终确认，确保下拉选项一致 | 高 |
| 联调验证 | 权限体系联调（多角色冲突、按钮禁用提示、路由守卫） | 中 |
| 功能补齐 | 导出功能：各列表页导出按钮当前降级显示"暂未开放"，待后端提供导出接口后启用 | 中 |
| 体验优化 | 空态 / 错误态细化；联调环境关闭 `VITE_USE_LIST_FALLBACK` | 低 |
| 部署对接 | 生产环境部署配置、接口变更与文档同步 | 中 |
